# Decision 002: Post Navigation (Prev / Next)

> 2026-04-17 — 블로그 포스트 하단에 이전/다음 포스트 네비게이션 추가.

---

## 배경

블로그 포스트를 다 읽은 후 다음 글로 이어서 읽을 수 있는 경로가 없었음. 일반적인 기술 블로그에서는 포스트 하단에 Prev/Next 네비게이션을 제공.

---

## 네비게이션 로직

### 우선순위

1. **시리즈 포스트** → 같은 시리즈 내에서만 이전/다음 (`seriesOrder` 기준)
2. **시리즈에 혼자뿐** (이전/다음 둘 다 없음) → 날짜순 폴백 (`pubDate` 기준)
3. **비시리즈 포스트** → 날짜순

### 왜 시리즈 우선인가

시리즈를 읽는 사람은 그 시리즈의 흐름을 따라가고 있음. Design to Code #2 다 읽고 "Next"를 눌렀는데 Solo Builder #1이 나오면 맥락이 끊김. 시리즈 끝에서 다른 시리즈로 넘어가는 폴백은 초기에 구현했다가 제거.

### 혼합 폴백 금지

시리즈 내 이전/다음이 한쪽만 있을 때 빈 쪽을 날짜순으로 채우지 않음. 예: Design to Code #2는 `← #1` 만 표시하고 Next는 없음 (#3이 아직 없으므로). 이렇게 해야 시리즈의 끝이 명확함.

단, 시리즈에 포스트가 1개뿐이면 (이전/다음 모두 없음) 날짜순 폴백 적용. 네비게이션이 아예 없는 것보다 낫기 때문.

---

## 디자인

### 구조

```
[○← ] Prev                    Next [ ○→]
       제목...                 제목...
```

- 원형 화살표 버튼 (브랜드 보라색 테두리)
- Prev: 화살표 왼쪽 + 텍스트 오른쪽
- Next: 텍스트 왼쪽 + 화살표 오른쪽
- 은은한 카드 배경 (`bg-subtle`), 테두리 없음
- Hover: 화살표 원 채움 (보라색 bg + 흰색 아이콘), 카드 bg 약간 어두워짐

### 반응형

- **데스크탑**: 2열 그리드 (같은 높이)
- **모바일**: 1열, Next 위 + Prev 아래 (다음 글 우선 — 읽기 흐름상 자연스러움)

### 제목 처리

- 1줄 말줄임 (`text-overflow: ellipsis; white-space: nowrap`)
- grid item에 `min-width: 0` 필수 — 없으면 `white-space: nowrap`이 그리드 셀을 밀어내서 가로 스크롤 발생

### 라벨

- "Prev" / "Next" 고정 — 짧고 균일한 문자수
- "Previous in series" 같은 긴 라벨은 모바일에서 2줄 됨 → 금지

## 교차 배포 영향

**없음.** 네비게이션은 BlogPost.astro 레이아웃에만 존재. 교차 배포 스크립트(`publish-post.ts`)는 마크다운 본문만 추출하므로 레이아웃/CSS/네비게이션은 dev.to / Hashnode에 전송되지 않음.
